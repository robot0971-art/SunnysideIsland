# 내일 할일 - Campfire 상호작용 수정

## 문제 요약

Campfire 상호작용이 F키(또는 E키)로 작동하지 않음.

---

## 문제 원인

### 1. PlayerController가 IInteractable 인터페이스를 체크하지 않음

**현재 코드 (`PlayerController.TryInteract()`):**
```csharp
// _interactableLayer로 검출된 객체를 FarmPlot 타입만 체크
RaycastHit2D hit = Physics2D.Raycast(origin, _facingDirection, distance, _interactableLayer);
if (hit.collider != null && hit.collider.TryGetComponent(out FarmPlot plot))
{
    // FarmPlot 처리만 함 - IInteractable은 무시됨!
}
```

### 2. 상호작용 키 불일치 가능성

- 코드: `KeyCode.E` (E키)
- 사용자 시도: F키일 수 있음

---

## 해결 방안

### 방법 1: PlayerController에 IInteractable 체크 추가 (권장)

**파일**: `Assets/Scripts/Player/PlayerController.cs`

**수정 위치**: `TryInteract()` 메서드 (약 368-421행)

**수정 내용**:
```csharp
private bool TryInteract()
{
    Vector2 origin = (Vector2)transform.position + new Vector2(0, 0.1f);
    float distance = 1.5f;
    float radius = _interactionRadius;
    
    // 1. IInteractable 체크 (Campfire, NPC, 상자 등)
    RaycastHit2D interactableHit = Physics2D.Raycast(origin, _facingDirection, distance, _interactableLayer);
    if (interactableHit.collider != null)
    {
        // IInteractable 인터페이스 체크
        if (interactableHit.collider.TryGetComponent(out IInteractable interactable))
        {
            if (interactable.CanInteract())
            {
                interactable.Interact();
                return true;
            }
        }
        
        // 기존 FarmPlot 로직 유지
        if (interactableHit.collider.TryGetComponent(out FarmPlot plot))
        {
            // ... 기존 FarmPlot 처리 코드
            return true;
        }
    }
    
    // 2. harvestLayer 체크 (기존 코드 유지)
    // ...
    
    // 3. treeLayer 체크 (기존 코드 유지)
    // ...
    
    return false;
}
```

## 추가 확인 사항

### Campfire 레이어 설정

Campfire Prefab의 Layer가 `Interactable` 또는 `_interactableLayer`에 포함되어 있는지 확인:

1. `Assets/Prefabs/Campfire.prefab` 선택
2. Inspector에서 Layer 확인
3. `Interactable` 레이어로 설정 (또는 PlayerController의 `_interactableLayer`에 포함된 레이어)

---

## 레이어 vs 태그 인식 비교

| 항목 | 레이어 인식 | 태그 인식 |
|------|-----------|----------|
| **성능** | ✅ 빠름 (Physics 최적화) | ❌ 느림 (문자열 비교) |
| **유연성** | ✅ 여러 IInteractable 배치 가능 | ❌ 오브젝트당 하나의 태그만 가능 |
| **확장성** | ✅ 레이어만 맞추면 추가 가능 | ❌ 코드 수정 필요 |
| **Physics 필터링** | ✅ 가능 | ❌ 불가능 |

**결론**: **레이어 인식 + IInteractable 체크** 조합이 최선
